## 학습목표
구조체와 파일 입출력의 개념을 이해한다.
구조체 정의 방법을 알아보고 구조체 배열을 작성해본다.
파일 포인터를 이용하여 파일에 데이터를 저장하고 읽을 수 있다.
구조체와 파일 포인터를 이용하여 데이터를 읽어오고 구조체로 저장하거나, 
반대로 구조체 데이터를 저장하는 방법을 학습한다.

## 개념잡기
구조체는 서로 다른 데이터 타입을 하나의 묶음으로 정의하는 사용자 정의 데이터 타입이다.
파일 입출력은 프로그램에서 데이터를 파일에 저장하거나, 파일에서 데이터를 읽어오는 것을 말한다.
파일 포인터는 파일을 가리키는 포인터로, 파일에서 데이터를 읽거나 쓸 때 사용된다.

## 강의요약
파일에 구조체 변수를 저장하는 방법을 알아본다. 이를 위해서는 파일 포인터와 fwrite() 함수를 사용한다.
파일에서 구조체 변수를 읽어오는 방법을 알아본다. 이를 위해서는 파일 포인터와 fread() 함수를 사용한다.
구조체 변수를 파일에 저장할 때, 구조체의 멤버 변수들을 한번에 저장하기 위해 sizeof 연산자를 활용한다.
파일에 저장된 데이터를 출력하는 방법을 학습한다. 이를 위해서는 파일 포인터와 fread() 함수를 사용한다.

## 학습목표
1. 파일 입출력의 개념과 종류를 이해한다.
1. 파일을 생성하고 열고 닫는 방법을 익힌다.
1. 파일에 데이터를 쓰고 읽는 방법을 이해하고 실습한다.
1. 텍스트 파일과 이진 파일의 차이점을 이해하고 실습한다.

## 개념잡기
파일 입출력은 컴퓨터의 주기억장치와 보조기억장치 사이의 데이터 전송을 말한다. 파일 입출력은 주로 프로그램에서 사용하는 데이터를 파일에 저장하고, 나중에 다시 불러와 사용하는 용도로 많이 사용된다. 파일 입출력은 C언어에서도 지원되며, 파일 입출력을 위해서는 파일을 생성하고 열어서 파일에 데이터를 쓰거나 파일에서 데이터를 읽어와야 한다.

파일은 일반적으로 텍스트 파일과 이진 파일로 구분된다. 텍스트 파일은 사람이 읽을 수 있는 문자로 이루어진 파일로, 일반적으로 텍스트 편집기로 편집할 수 있다. 이진 파일은 바이너리 코드로 이루어진 파일로, 텍스트 편집기로 편집할 수 없다. 이진 파일은 주로 프로그램 실행 파일이나 이미지, 음악, 동영상 등의 멀티미디어 파일로 사용된다.

## 강의요약
**파일 생성하기**
파일을 생성하기 위해 fopen 함수를 사용한다.
파일 생성 모드에는 읽기 모드(r), 쓰기 모드(w), 추가 모드(a)가 있다.

**파일 열고 닫기**
파일을 열기 위해 fopen 함수를 사용한다.
파일을 닫기 위해 fclose 함수를 사용한다.

**파일에 데이터 쓰기**
파일에 데이터를 쓰기 위해 fprintf 함수를 사용한다.

**파일에서 데이터 읽기**
파일에서 데이터를 읽기 위해 fscanf 함수를 사용한다.

**이진 파일 다루기**
이진 파일은 바이너리 형식으로 구성되어 있기 때문에, 텍스트 파일과 달리 fread, fwrite 함수를 사용하여 데이터를 읽고 쓴다.

**예외 처리하기**
파일 입출력에서는 파일이 존재하지 않거나 열기/쓰기 권한이 없는 등의 예외 상황이 발생할 수 있다. 이러한 예외 상황을 처리하기 위해 에러 처리 함수를 사용한다.

## 구조체와 파일 포인터
해당 부분은 근본적인 이해보단 사용법에 집중하며 따라옵시다.
때문에 해당 부분은 원리보단 활용 위주로 학습을 해보려합니다.

## 구조체는
여러 속성을 갖는 객체는 일상생활에서 쉽게 볼 수 있습니다.
어렵게 표현을 했는데, 비유하면 쉽습니다.
"학교에서 학생 성적을 처리할 때 어떤 정보가 필요할까요?"
라고 했을 때 여러분들 머리속에는
1. 학생 이름
1. 학년
1. 반
1. 번호
1. 과목 점수 A
1. 과목 점수 B
정도 떠올렸을 것 같습니다.

즉, 학생이란 타입은 그 안에 여러 개의 데이터를 갖고 있습니다.
이를 표현하기위해 배열을 사용할 수도 있지만, 좀 더 세련된 방법으로 '구조체'를 사용하는 방법이 있습니다. 그리고 이런 상황이라면 '구조체'를 사용하는 게 더 적합한 경우입니다. c언어로는 아래처럼 정의할 수 있습니다.

```c
struct Student{
    char[20] name;
    int grade, class, number;
    int subjectA, subjectB;
}
```

## 파일 포인터는
포인터 변수를 사용하긴 할 텐데, 사실 파일을 조작하려면 반드시 포인터 변수를 사용해야합니다. 왜 그런지 생각해보죠.

1. 파일은 원본 하나가 있는 상태입니다.
1. 이곳의 데이터를 읽거나 새로운 데이터를 작성하려면
1. 원본에 접근 할 수 있는, 주소로 원본 데이터에 접근하고 다루는 포인터 변수를 사용해야합니다.

## 파일 접근 모드
파일 포인터 변수를 사용하기 전에 알아둬야할 모드 3가지가 있습니다.
오늘 실습에선 'r'과 'w'만 사용할 예정입니다.
|모드|모드설명|
|-|-|
| 'r'  | 읽기 전용으로 파일 열기 |
| 'w'  | 쓰기 전용으로 파일 열기 (파일이 존재하지 않으면 생성하고, 존재하면 파일 내용을 지우고 새로 씁니다.) |
| 'a'  | 쓰기 전용으로 파일 열기 (파일이 존재하지 않으면 생성하고, 존재하면 파일 끝에 새로운 내용을 추가합니다.) |
| 'r+' | 읽기 및 쓰기 모드로 파일 열기 |
| 'w+' | 읽기 및 쓰기 모드로 파일 열기 (파일이 존재하지 않으면 생성하고, 존재하면 파일 내용을 지우고 새로 씁니다.) |
| 'a+' | 읽기 및 쓰기 모드로 파일 열기 (파일이 존재하지 않으면 생성하고, 존재하면 파일 끝에 새로운 내용을 추가합니다.) |

## 파일 포인터 관련 함수
fputs, fgets보단 fprintf, fscanf를 추천합니다. 이유는 실습을 통해 알아봅시다.

|모드|모드설명|
|-|-|
| fopen() | 파일을 열기 위한 함수로, 파일의 경로와 열기 모드를 인자로 받습니다. 파일을 열 때 사용되는 열기 모드에는 읽기(r), 쓰기(w), 추가(a), 이진 모드(b) 등이 있습니다. |
| fclose() | 파일을 닫기 위한 함수로, fopen() 함수로 열린 파일을 닫아줍니다. 파일을 닫지 않으면 프로그램이 종료될 때까지 파일이 사용 중인 상태로 남아있을 수 있습니다. |
| fgets() | 파일로부터 한 줄씩 문자열을 읽어오는 함수로, 파일 포인터와 문자열을 저장할 버퍼, 그리고 버퍼의 크기를 인자로 받습니다. 파일의 끝에 도달하거나 버퍼의 크기를 초과할 때까지 문자열을 읽어옵니다. |
| fputs() | 파일에 문자열을 쓰는 함수로, 문자열과 파일 포인터를 인자로 받습니다. 파일에 문자열을 쓸 때는 개행 문자를 포함하여 쓸 수 있습니다. |
| fprintf() | 파일에 형식화된 출력을 쓰는 함수로, printf() 함수와 비슷한 형식으로 문자열과 값을 인자로 받습니다. 문자열 내에 %d, %f 등의 서식 지정자를 사용하여 값을 출력할 수 있습니다. |
| fscanf() | 파일로부터 형식화된 입력을 받는 함수로, fprintf() 함수와 비슷한 형식으로 문자열과 값을 인자로 받습니다. 파일에서 값을 읽어와서 변수에 저장합니다. |
