## 포인터(pointer)
발표할 때 화면을 가리키는 보조도구를 포인터라고 합니다.
누군가 손으로 무언가를 가리키면, 우리의 시선은 그 손가락 끝을 따라가 가리키는 물건 또는 건물을 보게됩니다.
이 때 손은 포인터라고 볼 수 있습니다.
포인터는 무언가를 가리키는 것. 그 무언가는 어떤 물건의 위치입니다.

## 프로그래밍에서, C언어에서의 포인터
우리는 작성한 코드들은 이미 포인터를 사용하고 있습니다.
언제부터였냐면 '변수'를 배울 때 부터요.
우리는 변수 이름을 통해 데이터에 접근할 수 있습니다.
변수 이름에는 데이터가 저장되어있는게 아니라, 데이터가 저장된 주소가 저장되있어요.
scanf 함수를 사용할 때를 생각해봅시다.

```c
int age;
printf("당신의 나이는? : ");
scanf("%d", &age);  // 입력받는 데이터의 포맷코드, 저장할 변수의 주소값
```
- 그냥 `변수이름`만 작성하면 데이터의 값을 다루게 되니
- 데이터의 주소값을 입력해주기위해 `&변수이름` 으로 값을 넣어주었습니다.

즉 변수는 이미 포인터라는 개념을 사용하고 있습니다.

## 오늘 배울 건 포인트 변수
기본적으로 그냥 변수이름만 사용할 땐 '데이터 값'을 관리하게 됩니다.
값이 아닌 데이터의 주소를 관리하고 싶을 때 사용하는 것이 '포인터 변수'입니다.

일반 수학 계산과 같은 경우에 '포인터 변수'는 사용할 필요가 크게 없어서,
C언어 기초 강의들을 보면 포인터를 뺀 경우도 있습니다. 
이해하기 위해 알아야할 배경 지식도 꽤 되고, 변수와 포인터 변수의 차이를 이해하지 못하여 어렵다고 느끼는 구간이거든요. 
실제로 포인터 변수 배우는 구간에서 C언어 배우는 것을 많이 포기합니다.

다만 이 강의를 보시는 분들은 아두이노와 같은 IoT나 임베디드 프로그래밍을 하시게 될 것으로 예상되는데,
이 경우 배열(array)와 같은 데이터 구조를 자주 사용하게 될 것이고,
이런 연속적인 데이터를 잘 사용하고 관리하려면 포인터 변수를 배울 필요가 있습니다.
최대한 필요한 개념을 단순하게 요약하고 시각적으로 이해하며 가보도록 하죠.

## 포인터는 입문자에게 어렵습니다. 다만 포인터를 100% 이해해야 개발할 수 있는 건 또 아닙니다.
여러분들이 해당 포인터 부분을 몰라도, 당장 그리고 꽤 오랫동안 프로그래밍을 하는데 무리는 없을 겁니다.

이 때문에 포인터를 어려워합니다. 왜 굳이 써야하지?란 의문 때문에 더 이해하기 어려워집니다. 포인터가 필요한 이유를 입문자가 다루는 로직, 데이터 크기에선 공감하기 어렵습니다.

다만 C언어의 장점을 살려 프로그래밍 하기 위해선 + 프로그래밍의 동작 방식을 이해하고 더 빠르고 효율적인 프로그램을 만들기 위해선, 포인터와 컴퓨터 메모리 주소에 대해 알아야합니다.

## 그래도 일상 생활에서 발견해보는 포인터의 용도 1가지
여러분이 놀이공원 중앙에서 안내 아르바이트를 하고 있습니다.

그러던 도중 아래 질문이 왔다고 합시다.
1. 디스코디스코 트위스트는 무엇인가요?
2. 원심분리기롤러코스터는 무엇인가요?

여기서 여러분은
1. 그 놀이기구를 몸 동작으로 어떻게 생겼고 크기는 얼마큼인지 정확하게 표현하려고 노력할 수도 있고
2. 그냥 손가락으로 '저거입니다.' '이거입니다.' 라고 할 수도 있습니다.

1은 설명하는 데 시간도 걸리고, 만일 놀이기구의 1:1 크기로 설명을 하려면 그만한 공간이 있어야 할 것입니다.
2는 그 손가락만 따라가면 바로 그 놀이기구를 알 수 있습니다. 놀이기구의 크기가 얼마나 하던지 상관없이, 다른 사람이 보고 눈길을 돌릴 수 있을 정도 크기의 손가락만 있으면 됩니다.

1은 포인터를 안쓰고 데이터를 복사, 참고해와서 사용하는 방식이라면, 2는 오늘 배울 포인터(손가락, 가리키기)를 이용한 방식입니다.

다른 곳에서 데이터를 복사하거나 참조할 때, 포인터를 쓸 때와 안 쓸때의 차이를 중간마다 떠올려보며 오늘 학습을 배워봅시다.

## 포인터 변수 선언 방법
```c
타입 *포인터이름 = 주소값;
타입* 포인터이름 = 주소값
타입 * 포인터이름 = 주소값

타입 *포인터이름 = &변수이름;
타입* 포인터이름 = &변수이름;
타입 * 포인터이름 = &변수이름;
```

## 포인터 변수의 특징
1. 데이터의 주소를 저장하게 됩니다.
1. 연속된 데이터를 포인터 변수로 저장할 경우, 1개 데이터 단위로 이동 및 연산을 하게 됩니다.
    - 때문에 연속된 데이터는 모두 동일 데이터 형이여야 합니다.
1. 데이터의 크기 단위만큼 이동+연산을 합니다.
    1. int형 배열을 포인터 변수로 가리키게 되면
    1. int형 사이즈 만큼이 이 포인터 변수 연산의 기본 단위가 됩니다.
